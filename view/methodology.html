Our Methodology

Approach of how we work.

Agile.

Software is developed in incremental, rapid cycles. This results in small incremental releases with each release building on previous functionality. Each release is thoroughly tested to ensure software quality is maintained.

Learn More

Kanban.

Kanban became an effective tool to support running a production system as a whole, and an excellent way to promote improvement. With a Kanban, you will easily gasp the statuses of tasks, so that the process of the sprint in totally under your control.

Daily Status
Updates.

Hwaztech developers report you their daily statuses everyday. It provides a reliable and easy way to communicate the status of particular project activities.

Typically, a daily status report consists of three parts:

  • What I have done today
  • What will I do tomorrow
  • Questions & Summary

Demo.

You will be given a demo by the end of each sprint to check if the implementation followed the initial design, and if the implementation is exactly what you want. So that you are able to execute any adjustments as earlier as possible.

Retrospective.

The Agile retrospective can be thought of as a "lessons learned" meeting. The team reflects on how everything went and then decides what changes they want to make in the next iteration. Typically, a retrospective consists of three parts:

  • What worked well for us?
  • What did not work well for us?
  • What actions can we take to improve our process going forward?

Code
Review.

Continuously improving the quality of the code and architecture, consistent design and implementation, teaching and sharing knowledge between team members.
